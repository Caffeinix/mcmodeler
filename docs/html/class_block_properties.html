Detailed Description

Provides access to the properties of a particular block type.

Most of this information is only of interest to Renderable subclasses, and is not exposed broadly. The rest can be acquired by consulting the BlockPrototype. BlockProperties is an immutable class with no non-const methods.

Constructor & Destructor Documentation

BlockProperties::BlockProperties ( )

Constructs an invalid BlockProperties instance.

Because BlockProperties is immutable, this is only useful as a placeholder (for example, in calls to QMap::value).

BlockProperties::BlockProperties ( const QVariantMap &  block_data)
explicit

Constructs a BlockProperties instance out of the given block_data map.

This map should contain keys which match the names of Q_PROPERTIES of this class. The spriteOffset property will be populated from a two-element list of integers, the tileOffsets property will be populated from a list of two-element lists of integers, and the validOrientations property will be populated from a list of strings.

Note:
Generally, the only place this constructor needs to be called is inside BlockPrototype::setupBlockProperties(). To get properties like name, transparency, and orientations for an existing block type, go through BlockPrototype instead.

QVector< BlockOrientation * > BlockProperties::validOrientations ( ) const

Returns a list of valid orientations for this block type.

The orientations should be ordered such that the first valid orientation is a reasonable default.

QVector< QPoint > BlockProperties::tileOffsets ( ) const

Returns a list of tile offsets within the terrain.png file.

The number and order of offsets in the list depends on the geometry. For a cube, there will be six offsets (one per face). The Renderable subclass is responsible for knowing how to interpret this information.

bool BlockProperties::isTransparent ( ) const

Returns whether this block is "transparent" in the Minecraft sense.

Transparent blocks may or may not be visibly translucent, but they permit the passage of light, do not hide blocks behind them, and will not suffocate players. For the purposes of MCModeler, most of these properties are ignored, but adjacent block culling is not performed on transparent blocks.

bool BlockProperties::isBiomeGrass ( ) const

Returns whether this block should be shaded as a biome-compliant grass texture.

If true, the BlockPrototype will automatically tint the appropriate textures with a grass-green color.

bool BlockProperties::isBiomeTree ( ) const

Returns whether this block should be shaded as a biome-compliant tree leaf texture.

If true, the BlockPrototype will automatically tint the appropriate textures with a leafy green color.

bool BlockProperties::isValid ( ) const

Returns whether this BlockProperties object is valid.

This will be false if it was created with the default constructor and true otherwise.
