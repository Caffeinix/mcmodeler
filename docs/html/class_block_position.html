Detailed Description

A lightweight integer-precision position class for blocks.

Because there can only be one block in a given square meter, it is more efficient and convenient to use integers for block coordinates (they hash correctly, can be compared without worrying about roundoff errors, etc). When the block is rendered, a (real-valued) QVector3D is required, and this can be generated by calling either cornerVector() or centerVector(). For efficiency, these vectors are not calculated until they are needed, so creating and copying BlockPositions is fairly cheap.
