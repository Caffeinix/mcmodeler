Detailed Description

Represents a possible valid orientation of a block.

BlockOrientations cannot be constructed directly. Instead, they are initialized by means of the get() method, which takes a uniquely identifying string, or the noOrientation() method which returns a default instance. This ensures that there will be exactly one BlockOrientation object per distinct string.

BlockOrientations are similar to enum constants in that they are useful purely for comparison to other BlockOrientations, but they are true objects instead of integers under the covers. This allows a descriptive name to be associated with each instance that can be retrieved by calling the name() method.

Member Function Documentation

BlockOrientation * BlockOrientation::noOrientation ( )
static

Returns a BlockOrientation object representing no orientation.

This is the default orientation for blocks like dirt, stone, and gravel that look the same from every direction, and for blocks like grass and snow that cannot be rotated out of their default orientation.

Note:
This function does not transfer ownership of the pointer to the caller, so you should not delete it!
BlockOrientation * BlockOrientation::paletteOrientation ( )
static

Returns a BlockOrientation object representing the orientation a block should be in when it is viewed in the block picker palette.

This is considered a unique orientation, even if it is visually identical to some other orientation.

Note:
This function does not transfer ownership of the pointer to the caller, so you should not delete it!
