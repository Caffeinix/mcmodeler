Detailed Description

Abstract class representing objects that can be rendered in 3D.

Each BlockPrototype has an associated Renderable, which is instantiated as a particular subclass depending on the block geometry. The textures for the block type are passed to the Renderable using setTexture(). Each Renderable subclass expects a certain number of textures, and renders them on the geometry it knows how to draw. The same Renderable object is used to draw every instance of a block type.

void Renderable::initialize ( )
virtual

Initializes this Renderable.

This must be called before the renderable can be used to draw geometry.

Remarks:
The need for an explicit initialize method is regrettable, but it is a consequence of the poor design of the C++ language: it is impossible to safely call virtual methods from a true constructor.

Reimplemented in BasicRenderable.

For certain operations (currently determining whether we should skip rendering a particular face due to its adjacency with a solid block), we need to consult a delegate with more information on the block we're rendering than we do.

This is usually the BlockPrototype itself, but Renderable doesn't know that.
